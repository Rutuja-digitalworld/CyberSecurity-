**DNS (Domain Name System)**<br>
What is DNS?<br>
➡DNS (Domain Name System) translates website names into IP addresses.<br>
It helps users access websites without remembering complex IP addresses.<br>
➡Example<br>
tryhackme.com  →  104.26.10.229<br>
DNS is the internet's phonebook. It converts domain names into IP addresses.<br>

**TLD (Top-Level Domain)**<br>
➡TLD is the rightmost part of a domain name.<br>
Example: In tryhackme.com, .com is the TLD.<br>
#Two types:<br>
➡gTLD (Generic TLD): .com, .org, .edu, .gov<br>
➡ccTLD (Country Code TLD): .in, .ca, .co.uk<br>
gTLDs indicate a website's purpose, while ccTLDs indicate a country or region.<br>

**Second-Level Domain**<br>
➡Taking tryhackme.com as an example, the .com part is the TLD, and tryhackme is the Second Level Domain. <br>
When registering a domain name, <br>
➡the second-level domain is limited to 63 characters + the TLD and can only use a-z 0-9 and hyphens (cannot start or end with hyphens or have consecutive hyphens).
<br>
**Subdomain**<br>
➡A subdomain appears before the Second-Level Domain (SLD).<br>
Example: In admin.tryhackme.com, admin is the subdomain.<br>
➡Rules:<br>
 •Maximum 63 characters<br>
 •Can use a-z, 0-9, and hyphens (-)<br>
 •Cannot start/end with a hyphen or have consecutive hyphens (--)<br>
 •Multiple subdomains can be used:<br>
Example: jupiter.servers.tryhackme.com<br>
 •Total domain length must be 253 characters or less.<br>
 •There is no limit on the number of subdomains.<br>

**DNS RECORD TYPES**
**A Record**
➡Resolves a domain name to an IPv4 address.
**AAAA Record**
➡Resolves a domain name to an IPv6 address.
**CNAME Record**
➡Points one domain/subdomain to another domain.
➡Example:
store.tryhackme.com → shops.shopify.com
**MX Record**
➡Specifies the email server for a domain.
➡Has a priority value for backup mail servers.
**TXT Record**
➡Stores text data.
➡Common uses:
➡Domain ownership verification
➡SPF (email authentication)
➡DMARC (email security)

**What Happens When You Make a DNS Request?**
**Steps**
1.Local Cache Check
➡Your computer first checks its local DNS cache.
➡If the IP is found, the process ends.
2.Recursive DNS Server
➡If not found, the request is sent to the Recursive DNS Server (usually provided by the ISP).
➡It also checks its cache.
3.Root DNS Server
➡If no cached result exists, the request goes to a Root DNS Server.
➡The root server identifies the TLD (.com, .org, .in) and redirects the request.
4.TLD Server
➡The TLD server tells where the domain's Authoritative DNS Server (Nameserver) is located.
5.Authoritative DNS Server
➡Stores the actual DNS records of the domain.
➡Returns the requested record (A, AAAA, MX, etc.).
6.Response Returned
➡The result is sent back to:
➡Recursive DNS Server
➡Your Computer
7.Caching
➡The result is stored temporarily using TTL (Time To Live).
➡TTL is the time (in seconds) for which a DNS record remains cached.
