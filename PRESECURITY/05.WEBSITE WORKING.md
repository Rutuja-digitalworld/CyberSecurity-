When you visit a website, your browser (like Safari or Google Chrome) makes a request to a web server asking for information about the page you're visiting. It will respond with data that your browser uses to show you the page; a web server is just a dedicated computer somewhere else in the world that handles your requests.<br>

**here are two major components that make up a website:**<br>
➡Front End (Client-Side) - the way your browser renders a website.<br>
➡Back End (Server-Side) - a server that processes your request and returns a response.<br>

**HTML (HyperText Markup Language)**<br>
➡HTML is used to create the structure of web pages.<br>
➡CSS is used for styling and appearance.<br>
➡JavaScript adds interactivity and dynamic features.<br>

**JavaScript (JS)**<br>
➡JavaScript (JS) is a programming language used to make web pages interactive.<br>
➡HTML provides structure, CSS provides styling, and JavaScript provides functionality.<br>
➡JS allows web pages to update content dynamically without reloading.<br>
➡Used for button clicks, form validation, animations, pop-ups, and real-time updates.<br>
➡Without JavaScript, websites would be mostly static (non-interactive).<br>

**HTML Injection**<br>
➡HTML Injection is a vulnerability where an attacker inserts HTML code into a webpage through unsanitized user input.<br>
➡It occurs when a website displays user input without proper filtering (sanitization).<br>
➡The injected HTML is executed/rendered by the browser.<br>
➡Attackers can modify the page's appearance or functionality.<br>
➡HTML Injection is a client-side vulnerability.<br>
**Input Sanitization**<br>
➡Input Sanitization = Filtering user input to remove harmful HTML, JavaScript, or special characters.<br>
➡Prevents attackers from injecting malicious code.<br>

**Website Loading Process (Summary)**
1.User enters a website URL (e.g., tryhackme.com).
2.DNS converts the domain name into an IP address.
3.The browser connects to the web server using HTTP/HTTPS.
4.The web server sends back website files:
➡HTML → Structure
➡CSS → Styling
➡JavaScript → Functionality
➡Images/Videos → Media content
5.The browser processes these files and displays the webpage.

**Web Infrastructure - Quick Notes**
1. Load Balancer
•Used when a website receives high traffic or needs high availability.
•Receives user requests and forwards them to one of multiple servers.
•Distributes traffic among servers.
•Load Balancing Algorithms
Round Robin → Sends requests to each server in turn.
Weighted → Sends requests to the least busy server.
**Health Check**
•Periodically checks if servers are working.
•Stops sending traffic to unresponsive servers until they recover.
**Benefits**
✅ Handles high traffic
✅ Provides failover if a server goes down

**2. CDN (Content Delivery Network)**
•Stores static content on servers worldwide.
•Static files include:
➡JavaScript
➡CSS
➡Images
➡Videos
**How it Works**
•Sends users to the nearest CDN server.
•Reduces loading time and server traffic.
**Benefits**
✅ Faster website loading
✅ Reduced load on main server

**3. Databases**
•Used to store and retrieve website data.
•Web servers communicate with databases for user information.
•Common Databases
➡MySQL
➡MSSQL
➡MongoDB
➡PostgreSQL
**Benefits**
✅ Data storage
✅ Data retrieval
✅ Speed and reliability

**4. WAF (Web Application Firewall)**
•Sits between the user and the web server.
•Protects websites from attacks.
•Functions
➡Detects common hacking attempts.
➡Identifies bots vs real users.
➡Uses Rate Limiting to restrict requests from an IP address.
➡Rate Limiting
➡Allows only a certain number of requests per second from an IP.
•Action
➡Suspicious requests are blocked before reaching the server.
**Benefits**
✅ Protection from attacks
✅ Prevents DoS/DDoS attempts
✅ Filters malicious traffic

**Web Servers - Quick Notes**
*What is a Web Server?*
➡A Web Server is software that receives HTTP/HTTPS requests and delivers web content to users.
➡Common web servers:
•Apache
•Nginx
•IIS
•NodeJS
•Root Directory
➡Web files are stored in a root directory.
**Default locations:**
➡Linux (Apache/Nginx): /var/www/html
➡Windows (IIS): C:\inetpub\wwwroot
**Example**
*URL Requested:*
http://example.com/picture.jpg
*Server Returns:*
 /var/www/html/picture.jpg

 **Virtual Hosts**
•Allow a single web server to host multiple websites.
•Server checks the Host header and serves the matching website.
•If no match is found, the default website is shown.
*Example*
one.com → /var/www/website_one
two.com → /var/www/website_two
**Important Point**
✅ No limit on the number of websites a server can host

**Static vs Dynamic Content**
**Static Content**
•Content that never changes.
•Directly served by the web server.
•Examples
➡HTML pages
➡Images
➡CSS
➡JavaScript
**Dynamic Content**
•Content that changes based on user requests.
•Generated by the Backend.
•Examples
➡Blog posts
➡Search results
➡User dashboards
