ASCII stands for American Standard Code for Information Interchange, and it is an early character encoding from 1963 that uses numbers 0-127 to represent English letters, digits, punctuation, and some control characters.

# European Language Encodings – Quick Notes
•ASCII (7-bit) supports only English characters.

•Using 8 bits adds 128 more characters, but that's still not enough for all European languages.

•ISO/IEC 8859 standards were created to support different European languages.

*Important Standards*

Standard	                                 Supports

•ISO-8859-1 (Latin-1) =	Western European languages: German (ß, ü), French (é), Spanish (ñ, ¿), Italian, Portuguese, etc.

•ISO-8859-2 (Latin-2) =	Central/Eastern European languages: Polish (ł), Czech (č), Hungarian (ő), Romanian (ș), Slovak, etc.

**Key Point**

⚠️ If a file is saved in one encoding (e.g., Latin-1) and opened as another (e.g., Latin-2), special characters may appear incorrectly.

## Unicode & UTF – Quick Revision Notes
•Problem with ASCII & Extended ASCII

•ASCII (7-bit) supports only English characters (128 characters).

•Extended ASCII (8-bit) added more characters but different standards caused compatibility issues.

## Unicode
•Universal character encoding standard.
•Gives every character a unique code point.
•Supports all languages and emojis.
•Defines ~157,000 characters
•Includes ~4,000 emoji sequences

# UTF Encodings

Unicode characters are stored using UTF formats.

### UTF-8
•Most common on the web.
•Uses 1–4 bytes per character.
•Backward compatible with ASCII.

## UTF-16
•Uses 2 or 4 bytes.
•Common characters → 2 bytes.
•Emoji/rare characters → 4 bytes.

## UTF-32
•Uses exactly 4 bytes for every character.
•Simplest but wastes memory.
