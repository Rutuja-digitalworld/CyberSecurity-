**HTTP & HTTPS**<br>
*HTTP (HyperText Transfer Protocol)**<br>
→HTTP is the protocol used to communicate with web servers and load websites.<br>
→It transfers web content such as HTML, images, videos, and other files.<br>
→Developed by Tim Berners-Lee between 1989–1991.<br>
→Data sent using HTTP is not encrypted.<br>
Example
http://example.com<br>

*HTTPS (HyperText Transfer Protocol Secure)*<br>
→HTTPS is the secure version of HTTP.<br>
→Data is encrypted, preventing others from reading intercepted information.<br>
→Ensures you are communicating with the real website and not an impersonator.<br>
Example
https://example.com<br>

**URL (Uniform Resource Locator)**<br>
→A URL is the address used to access a resource on the internet.<br>
→It tells the browser what resource to access and how to access it.<br>

**Important Ports**<br>
Protocol	Default Port<br>
→HTTP	80<br>
→HTTPS	443<br>
Port Range: 1 - 65535<br>
<img width="1140" height="270" alt="URL" src="https://github.com/user-attachments/assets/76c4dc2f-e78b-4556-b98b-97fe771b12c1" /><br>
**Scheme:** This instructs on what protocol to use for accessing the resource such as HTTP, HTTPS, FTP (File Transfer Protocol).<br>

**User:** Some services require authentication to log in, you can put a username and password into the URL to log in.<br>

**Host:** The domain name or IP address of the server you wish to access.<br>

**Port:** The Port that you are going to connect to, usually 80 for HTTP and 443 for HTTPS, but this can be hosted on any port between 1 - 65535.<br>

**Path:** The file name or location of the resource you are trying to access.<br>

**Query String:** Extra bits of information that can be sent to the requested path. <br>
For example, /blog?id=1 would tell the blog path that you wish to receive the blog article with the id of 1.<br>

**Fragment:** This is a reference to a location on the actual page requested.<br>
This is commonly used for pages with long content and can have a certain part of the page directly linked to it, 
so it is viewable to the user as soon as they access the page.<br>

*HTTP methods are a way for the client to show their intended action when making an HTTP request. <br>
There are a lot of HTTP methods but we'll cover the most common ones, although mostly you'll deal with the GET and POST method.*<br>

**GET Request**<br>
This is used for getting information from a web server.<br>

**POST Request**<br>
This is used for submitting data to the web server and potentially creating new records<br>

**PUT Request**<br>
This is used for submitting data to a web server to update information<br>

**DELETE Request**<br>
This is used for deleting information/records from a web server.<br>

**Cookies**<br>
→Cookies are small pieces of data stored in a user's browser.<br>
→They are created when a server sends a Set-Cookie header.<br>
→The browser sends the cookie back to the server with future requests.<br>
**Why are Cookies Used?**<br>
→Remember user login sessions<br>
→Store website preferences<br>
→Track whether a user has visited before<br>
→Maintain user authentication<br>
**Important Point**<br>
→HTTP is stateless, meaning it does not remember previous requests.<br>
→Cookies help websites remember users between requests.<br>
**Authentication Cookies**<br>
→Cookies usually store a token (unique secret code), not the actual password.<br>
→This token helps identify and authenticate the user.<br>
