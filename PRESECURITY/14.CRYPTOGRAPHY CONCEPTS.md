# Cryptography 

•The science of securing data by keeping it secret, authentic, and unchanged.

•It is one of the most critical tools in a defender's arsenal.

•It's one layer in a much bigger security picture that includes:

✅Strong password practices.

✅Secure key storage.

✅User awareness and training.

✅Regular software updates.

✅Monitoring and incident response.

## SYMMETRIC ENCRYPTION
one key locks the box, the same key unlocks it.

Features

✅ Same key locks and unlocks data.

✅ Both sender and receiver need the same secret key.

✅ Fast and efficient for large amounts of data

•**Plaintext** - A message you can read normally. Like HELLO or Patient name: Alice Smith.

•**Ciphertext** - A scrambled version that's not supposed to make sense. Like KHOOR or Sdwlhqw qdph: Dolfh Vplwk.

•**Key** - The secret ingredient that controls how scrambling and unscrambling work. Think of it as a password that the algorithm uses.

•**Algorithm** - The public recipe—the set of steps that explain how to use the key on the message. Everyone can know the algorithm. Security comes from keeping the key secret.

 ## Lockbox Analogy (Cryptography) 🔐
 
•**Algorithm** → Method/rules used for encryption (like how the lock works).

•**Key** → Secret code needed to lock/unlock data.

•**Plaintext** → Original readable data/message.

•**Ciphertext** → Encrypted, unreadable data after encryption.

# Caesar Cipher
•**Algorithm:** Shift letters by a fixed number (public).

•**Key:** Secret shift value (e.g., 3).

•**Example:** HELLO → KHOOR (shift = 3).

## The Caesar Cipher: HOW IT WORKS

Encrypt HELLO with a key of 3:

H → K

E → H

L → O

L → O

O → R

So HELLO becomes KHOOR.

To decrypt KHOOR, Bob shifts each letter backwards by 3:

K → H

H → E

O → L

O → L

R → O

### Important

•Caesar Cipher is not secure and is used only for learning.

•Modern systems use strong algorithms like AES (Advanced Encryption Standard).

### Formula: Algorithm + Key + Plaintext → Ciphertext

### Uses
•File encryption

•Hard drive encryption

•Secure network traffic

•Drawback (Key Distribution Problem)

## How do you safely share the secret key? ❌

•If the key is intercepted, all messages can be decrypted.

•Encrypting the key would require another key → endless cycle.

## Asymmetric Encryption (Public Key Cryptography)🔑
**Definition:** initiates a connection and securely shares a symmetric key.

### Uses 2 mathematically linked keys:

**Public Key** → Shared with everyone.

**Private Key** → Kept secret by the owner.

### How It Works

✅ Encrypt with Public Key → Decrypt with Private Key

✅ Encrypt with Private Key → Verify using Public Key (Digital Signatures)

## Mailbox Analogy 📮

•*Mail Slot* = Public Key (anyone can use it)

•*Mailbox Key* = Private Key (only owner has it)

•Anyone can put a letter in, but only the owner can open and read it.

•Solves the Key Distribution Problem

**EXAMPLE**

•Bob creates Public + Private Key.

•Bob shares Public Key openly.

•Alice encrypts message using Bob's Public Key.

•Bob decrypts using his Private Key.

➡️ No need to secretly exchange keys.

### Advantages

✅ Solves key-sharing problem.

✅ More secure for communication setup.

### Disadvantages

❌ Slower than symmetric encryption.

### 🌐 HTTPS (Real-World Use)

When you visit a secure website:

1.Browser requests website's Public Key.

2.Website sends Certificate + Public Key.

3.Browser verifies certificate.

4.Browser & website create a shared symmetric key.

5.Further communication uses fast symmetric encryption.

### Hybrid Approach

•Asymmetric Encryption → Secure key exchange.

•Symmetric Encryption → Fast data transfer.

## 📜 Digital Certificate

A certificate:

•Contains the Public Key.

•Identifies the website/domain.

•Is signed by a trusted CA (Certificate Authority).

**Browser Checks**

✅ Certificate signed by trusted CA.

✅ Certificate not expired/revoked.

✅ Domain matches certificate.
