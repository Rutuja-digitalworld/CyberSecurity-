**DNS (Domain Name System)**<br>
What is DNS?<br>
➡DNS (Domain Name System) translates website names into IP addresses.<br>
It helps users access websites without remembering complex IP addresses.<br>
➡Example<br>
tryhackme.com  →  104.26.10.229<br>
DNS is the internet's phonebook. It converts domain names into IP addresses.<br>

**TLD (Top-Level Domain)**<br>
➡TLD is the rightmost part of a domain name.<br>
Example: In tryhackme.com, .com is the TLD.<br>
#Two types:<br>
➡gTLD (Generic TLD): .com, .org, .edu, .gov<br>
➡ccTLD (Country Code TLD): .in, .ca, .co.uk<br>
gTLDs indicate a website's purpose, while ccTLDs indicate a country or region.<br>

**Second-Level Domain**<br>
➡Taking tryhackme.com as an example, the .com part is the TLD, and tryhackme is the Second Level Domain. <br>
When registering a domain name, <br>
➡the second-level domain is limited to 63 characters + the TLD and can only use a-z 0-9 and hyphens (cannot start or end with hyphens or have consecutive hyphens).
<br>
**Subdomain**<br>
➡A subdomain appears before the Second-Level Domain (SLD).<br>
Example: In admin.tryhackme.com, admin is the subdomain.<br>
➡Rules:<br>
 •Maximum 63 characters<br>
 •Can use a-z, 0-9, and hyphens (-)<br>
 •Cannot start/end with a hyphen or have consecutive hyphens (--)<br>
 •Multiple subdomains can be used:<br>
Example: jupiter.servers.tryhackme.com<br>
 •Total domain length must be 253 characters or less.<br>
 •There is no limit on the number of subdomains.<br>

**DNS RECORD TYPES**<br>
**A Record**<br>
➡Resolves a domain name to an IPv4 address.<br>
**AAAA Record**<br>
➡Resolves a domain name to an IPv6 address.<br>
**CNAME Record**<br>
➡Points one domain/subdomain to another domain.<br>
➡Example:<br>
store.tryhackme.com → shops.shopify.com<br>
**MX Record**<br>
➡Specifies the email server for a domain.<br>
➡Has a priority value for backup mail servers.<br>
**TXT Record**<br>
➡Stores text data.<br>
➡Common uses:<br>
➡Domain ownership verification<br>
➡SPF (email authentication)<br>
➡DMARC (email security)<br>

**What Happens When You Make a DNS Request?**<br>
**Steps**<br>
1.Local Cache Check<br>
➡Your computer first checks its local DNS cache.<br>
➡If the IP is found, the process ends.<br>
2.Recursive DNS Server<br>
➡If not found, the request is sent to the Recursive DNS Server (usually provided by the ISP).<br>
➡It also checks its cache.<br>
3.Root DNS Server<br>
➡If no cached result exists, the request goes to a Root DNS Server.<br>
➡The root server identifies the TLD (.com, .org, .in) and redirects the request.<br>
4.TLD Server<br>
➡The TLD server tells where the domain's Authoritative DNS Server (Nameserver) is located.<br>
5.Authoritative DNS Server<br>
➡Stores the actual DNS records of the domain.<br>
➡Returns the requested record (A, AAAA, MX, etc.).<br>
6.Response Returned<br>
➡The result is sent back to:<br>
➡Recursive DNS Server<br>
➡Your Computer<br>
7.Caching<br>
➡The result is stored temporarily using TTL (Time To Live).<br>
➡TTL is the time (in seconds) for which a DNS record remains cached.<br>
