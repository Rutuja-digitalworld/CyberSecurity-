# SYSTEM CONFIGURATION 
The System Configuration utility (MSConfig) is for advanced troubleshooting, and its main purpose is to help diagnose startup issues. 

➡The utility has five tabs across the top. Below are the names for each tab. We will briefly cover each tab in this task. 

1.General

2.Boot

3.Services

4.Startup

5.Tools

➡**General Tab** – Selects which devices and services Windows loads during startup (Normal, Diagnostic, or Selective startup).

➡**Boot Tab** – Configures operating system boot options and startup settings.

* **Services Tab** – Displays all system services (running or stopped); services are background applications that run without user interaction.

**Startup Tab (MSConfig)** → Displays programs that automatically launch when Windows starts and lets you enable/disable startup apps to improve boot performance.

**Startup Management (Windows Server)**

* **MSConfig Startup Tab** → Not used for managing startup programs; Microsoft recommends using **Task Manager** instead.
* **Windows Server Difference** → Startup apps usually do **not appear** in Task Manager or MSConfig Startup tab.
* **View Startup Programs** → Press **Win + R → type `shell:startup` → Enter**.
* **Startup Folder** → Contains shortcuts/executables that run automatically when a user logs in.
* **Purpose** → Used to verify and manage user-level startup applications on Windows Server.

### MSConfig & Advanced System Settings (Short Notes)

* **Tools Tab (MSConfig)** → Lists Windows utilities/tools with brief descriptions for system configuration.

### Advanced System Settings

* **Advanced System Settings** → Extra options to manage performance and system recovery.
* **Performance Settings** → Controls system performance and virtual memory (page file).
* **Page File** → Uses disk space as extra RAM when physical memory is full.
* **Page File Details** → Shows drive location, initial size, maximum size, and whether Windows manages it automatically.
* **Startup and Recovery** → Configures system startup behavior and recovery options after crashes.
* **Crash Dump File** → Stores error details after a system crash (e.g., BSOD) for troubleshooting and analysis.
