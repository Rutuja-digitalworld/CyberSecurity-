# 1.Windows Command Prompt (CMD) – Short Notes

## What is CMD?

* **Command Prompt (cmd.exe)** is the default **Command-Line Interface (CLI)** in Windows.
* Used to interact with the operating system by typing commands instead of using a GUI.

## CLI vs GUI

### GUI (Graphical User Interface)

* Easy to learn and use.
* Intuitive with icons, buttons, and menus.
* Best for beginners.

### CLI (Command-Line Interface)

* Faster and more efficient once mastered.
* No need for multiple mouse clicks.
* Ideal for repetitive and administrative tasks.

## Advantages of CLI

* **Lower Resource Usage:** Uses less CPU and RAM than GUI.
* **Automation:** Easy to automate tasks using batch files or scripts.
* **Remote Management:** Easily manage remote systems using **SSH**.
* **Faster Workflow:** Execute commands directly without navigating menus.

## Learning Objectives

Using CMD, you can:

* Display basic system information.
* Check and troubleshoot network configuration.
* Manage files and folders.
* View and manage running processes.

## Remote Access (SSH)

* Connect to a remote Windows machine using:

  ```bash
  ssh user@MACHINE_IP
  ```
* First connection → Type **yes** to trust the host.
* Enter the password when prompted.

## Important Exam Point
* **Default Windows Command-Line Interpreter:** `cmd.exe` ✅

# 2.Windows CMD – Basic System Information (Short Notes)

### **Windows PATH**

* `set` → Displays **environment variables**.
* `Path=` shows directories where Windows searches for executable commands.

### **Important Commands**

| **Command**           | **Purpose**                                                          |
| --------------------- | -------------------------------------------------------------------- |
| `set`                 | Displays environment variables (including PATH).                     |
| `ver`                 | Shows the Windows OS version.                                        |
| `systeminfo`          | Displays detailed system information (OS, hostname, CPU, RAM, etc.). |
| `driverquery`         | Lists all installed device drivers.                                  |
| `driverquery \| more` | Displays driver list page by page.                                   |
| `help`                | Shows help information for commands.                                 |
| `cls`                 | Clears the Command Prompt screen.                                    |

### **Useful Tricks**

* Use **`| more`** to view long outputs one page at a time.
* Press **Spacebar** → Next page.
* Press **Ctrl + C** → Exit the output.

# 3.Windows CMD – Networking Commands (Short Notes)

## Network Configuration

| **Command**     | **Purpose**                                                                      |
| --------------- | -------------------------------------------------------------------------------- |
| `ipconfig`      | Displays IP address, Subnet Mask, and Default Gateway.                           |
| `ipconfig /all` | Shows complete network configuration (MAC address, DNS, DHCP, Lease info, etc.). |

---

## Network Troubleshooting

| **Command**        | **Purpose**                                                      |
| ------------------ | ---------------------------------------------------------------- |
| `ping <target>`    | Tests connectivity to another host and measures response time.   |
| `tracert <target>` | Displays the path (routers/hops) taken to reach the destination. |

---

## DNS Lookup

| **Command**                      | **Purpose**                                                    |
| -------------------------------- | -------------------------------------------------------------- |
| `nslookup <domain>`              | Finds the IP address of a domain using the default DNS server. |
| `nslookup <domain> <DNS Server>` | Uses a specific DNS server for lookup (e.g., `1.1.1.1`).       |

---

## Network Connections

| **Command**     | **Purpose**                                                                            |
| --------------- | -------------------------------------------------------------------------------------- |
| `netstat`       | Shows active network connections.                                                      |
| `netstat -a`    | Displays all active connections and listening ports.                                   |
| `netstat -b`    | Shows the executable/program using each connection.                                    |
| `netstat -o`    | Displays the Process ID (PID) for each connection.                                     |
| `netstat -n`    | Shows numerical IP addresses and port numbers.                                         |
| `netstat -abon` | Shows all connections, listening ports, executable names, PIDs, and numeric addresses. |

---

## Important Networking Terms

* **IPv4 Address** → Device's IP address on the network.
* **Subnet Mask** → Defines the network and host portions of an IP.
* **Default Gateway** → Router used to access other networks.
* **DNS Server** → Resolves domain names into IP addresses.
* **DHCP** → Automatically assigns IP addresses.
* **MAC Address (Physical Address)** → Unique hardware address of the network adapter.
* **TTL (Time To Live)** → Limits packet lifetime in a network.

---

## ⭐ Important Exam Points

* **MAC Address:** `ipconfig /all`
* **Check Connectivity:** `ping`
* **Trace Network Route:** `tracert`
* **DNS Lookup:** `nslookup`
* **View Network Connections:** `netstat`
* **Program using a Port:** `netstat -b`
* **Process ID (PID):** `netstat -o`
