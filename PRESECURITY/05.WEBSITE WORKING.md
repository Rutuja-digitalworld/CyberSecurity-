When you visit a website, your browser (like Safari or Google Chrome) makes a request to a web server asking for information about the page you're visiting. It will respond with data that your browser uses to show you the page; a web server is just a dedicated computer somewhere else in the world that handles your requests.<br>

**here are two major components that make up a website:**<br>
➡Front End (Client-Side) - the way your browser renders a website.<br>
➡Back End (Server-Side) - a server that processes your request and returns a response.<br>

**HTML (HyperText Markup Language)**<br>
➡HTML is used to create the structure of web pages.<br>
➡CSS is used for styling and appearance.<br>
➡JavaScript adds interactivity and dynamic features.<br>

**JavaScript (JS)**<br>
➡JavaScript (JS) is a programming language used to make web pages interactive.<br>
➡HTML provides structure, CSS provides styling, and JavaScript provides functionality.<br>
➡JS allows web pages to update content dynamically without reloading.<br>
➡Used for button clicks, form validation, animations, pop-ups, and real-time updates.<br>
➡Without JavaScript, websites would be mostly static (non-interactive).<br>

**HTML Injection**<br>
➡HTML Injection is a vulnerability where an attacker inserts HTML code into a webpage through unsanitized user input.<br>
➡It occurs when a website displays user input without proper filtering (sanitization).<br>
➡The injected HTML is executed/rendered by the browser.<br>
➡Attackers can modify the page's appearance or functionality.<br>
➡HTML Injection is a client-side vulnerability.<br>
**Input Sanitization**<br>
➡Input Sanitization = Filtering user input to remove harmful HTML, JavaScript, or special characters.<br>
➡Prevents attackers from injecting malicious code.<br>

**Website Loading Process (Summary)**<br>
1.User enters a website URL (e.g., tryhackme.com).<br>
2.DNS converts the domain name into an IP address.<br>
3.The browser connects to the web server using HTTP/HTTPS.<br>
4.The web server sends back website files:<br>
➡HTML → Structure<br>
➡CSS → Styling<br>
➡JavaScript → Functionality<br>
➡Images/Videos → Media content<br>
5.The browser processes these files and displays the webpage.<br>

**Web Infrastructure - Quick Notes**<br>
1. Load Balancer<br>
•Used when a website receives high traffic or needs high availability.<br>
•Receives user requests and forwards them to one of multiple servers.<br>
•Distributes traffic among servers.<br>
•Load Balancing Algorithms<br>
Round Robin → Sends requests to each server in turn.<br>
Weighted → Sends requests to the least busy server.<br>
**Health Check**<br>
•Periodically checks if servers are working.<br>
•Stops sending traffic to unresponsive servers until they recover.<br>
**Benefits**<br>
✅ Handles high traffic<br>
✅ Provides failover if a server goes down<br>

**2. CDN (Content Delivery Network)**<br>
•Stores static content on servers worldwide.<br>
•Static files include:<br>
➡JavaScript<br>
➡CSS<br>
➡Images<br>
➡Videos<br>
**How it Works**<br>
•Sends users to the nearest CDN server.<br>
•Reduces loading time and server traffic.<br>
**Benefits**<br>
✅ Faster website loading<br>
✅ Reduced load on main server<br>

**3. Databases**<br>
•Used to store and retrieve website data.<br>
•Web servers communicate with databases for user information.<br>
•Common Databases<br>
➡MySQL<br>
➡MSSQL<br>
➡MongoDB<br>
➡PostgreSQL<br>
**Benefits**<br>
✅ Data storage<br>
✅ Data retrieval<br>
✅ Speed and reliability<br>

**4. WAF (Web Application Firewall)**<br>
•Sits between the user and the web server.<br>
•Protects websites from attacks.<br>
•Functions<br>
➡Detects common hacking attempts.<br>
➡Identifies bots vs real users.<br>
➡Uses Rate Limiting to restrict requests from an IP address.<br>
➡Rate Limiting<br>
➡Allows only a certain number of requests per second from an IP.<br>
•Action<br>
➡Suspicious requests are blocked before reaching the server.<br>
**Benefits**<br>
✅ Protection from attacks<br>
✅ Prevents DoS/DDoS attempts<br>
✅ Filters malicious traffic<br>

**Web Servers - Quick Notes**<br>
*What is a Web Server?*<br>
➡A Web Server is software that receives HTTP/HTTPS requests and delivers web content to users.<br>
➡Common web servers:<br>
•Apache<br>
•Nginx<br>
•IIS<br>
•NodeJS<br>
•Root Directory<br>
➡Web files are stored in a root directory.<br>
**Default locations:**<br>
➡Linux (Apache/Nginx): /var/www/html<br>
➡Windows (IIS): C:\inetpub\wwwroot<br>
**Example**<br>
*URL Requested:*
http://example.com/picture.jpg
*Server Returns:*<br>
 /var/www/html/picture.jpg<br>

 **Virtual Hosts**<br>
•Allow a single web server to host multiple websites.<br>
•Server checks the Host header and serves the matching website.<br>
•If no match is found, the default website is shown.<br>
*Example*<br>
one.com → /var/www/website_one<br>
two.com → /var/www/website_two<br>
**Important Point**<br>
✅ No limit on the number of websites a server can host<br>

**Static vs Dynamic Content**<br>
**Static Content**<br>
•Content that never changes.<br>
•Directly served by the web server.<br>
•Examples<br>
➡HTML pages<br>
➡Images<br>
➡CSS<br>
➡JavaScript<br>
**Dynamic Content**<br>
•Content that changes based on user requests.<br>
•Generated by the Backend.<br>
•Examples<br>
➡Blog posts<br>
➡Search results<br>
➡User dashboards<br>
