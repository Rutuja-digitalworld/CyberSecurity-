# 1.Cryptography

* **Cryptography** = Practice of securing communication and protecting data from attackers.
* Ensures secure communication even in the presence of adversaries.
* Prevents unauthorized users from **reading** or **modifying** data.

---

# Objectives of Cryptography

* 🔒 **Confidentiality** – Keeps data secret from unauthorized users.
* ✔️ **Integrity** – Ensures data is not altered during transmission or storage.
* ✅ **Authenticity** – Verifies the identity of the sender and confirms data is genuine.

---

# Real-Life Uses of Cryptography

* **Website Login (HTTPS):** Encrypts usernames and passwords.
* **SSH:** Creates an encrypted tunnel for secure remote access.
* **Online Banking:** Verifies the bank's identity using digital certificates.
* **File Download:** Uses **hash functions** to verify file integrity.

---

# Data Protection States

* **Data at Rest:** Data stored on disks/databases → **Must be encrypted.**
* **Data in Motion (In Transit):** Data being transmitted over a network → **Must be encrypted.**

---

# Security Standards & Regulations

### Payment Card Data

* **PCI DSS (Payment Card Industry Data Security Standard)**

  * Protects credit/debit card information.
  * Requires encryption for **stored** and **transmitted** data.

### Medical Records

* **HIPAA** *(USA)* – Health Insurance Portability and Accountability Act.
* **HITECH** *(USA)* – Health Information Technology for Economic and Clinical Health.
* **GDPR** *(EU)* – General Data Protection Regulation.
* **DPA** *(UK)* – Data Protection Act.

# 2.Caesar Cipher

* One of the **oldest encryption techniques** (1st century BCE).
* Invented by **Julius Caesar**.
* A **substitution cipher** where each letter is shifted by a fixed number (**key**).

---

## How It Works

* **Encryption:** Shift each letter **right** by the key.
* **Decryption:** Shift each letter **left** by the same key.

**Example**

```text
Plaintext : TRYHACKME
Key       : 3
Ciphertext: WUBKDFNPH
```

---

## Components

* **Plaintext:** Original readable message.
* **Ciphertext:** Encrypted message.
* **Key:** Number of positions each letter is shifted.
* **Cipher:** Caesar Cipher algorithm.

---

## Security

* Only **25 possible keys** (26th shift returns the original text).
* Easily broken using a **Brute Force Attack** by trying all possible keys.
* **Not secure** by modern cryptographic standards.

# 3.Types of Encryption

There are **2 main types of encryption:**

1. **Symmetric Encryption**
2. **Asymmetric Encryption**

---

# 1. Symmetric Encryption (Private Key Cryptography)

* Uses the **same key** for **encryption** and **decryption**.
* Also called **Private Key Cryptography**.
* The shared key **must remain secret**.
* Fast and efficient for encrypting large amounts of data.
* Biggest challenge: **Securely sharing the secret key**.

### Working



### Examples

* **DES** – Data Encryption Standard (56-bit key) ❌ *Not Secure*
* **3DES** – Triple DES (168-bit key, ~112-bit effective security) ⚠️ *Deprecated (2019)*
* **AES** – Advanced Encryption Standard ✅ *(Current Standard)*

### AES Key Sizes

* 128-bit
* 192-bit
* 256-bit

---

# 2. Asymmetric Encryption (Public Key Cryptography)

* Uses **two different keys**:

  * **Public Key** → Encrypts data
  * **Private Key** → Decrypts data
* Also called **Public Key Cryptography**.
* **Public Key** can be shared with anyone.
* **Private Key** must always remain secret.
* More secure for key exchange but **slower** than symmetric encryption.

### Working



### Examples

* **RSA** (Recommended: **2048-bit** minimum)
* **Diffie-Hellman**
* **ECC (Elliptic Curve Cryptography)**

### Key Sizes

* **RSA:** 2048, 3072, 4096 bits
* **Diffie-Hellman:** 2048+ bits
* **ECC:** 256-bit ≈ 3072-bit RSA security

---

# Comparison

| Symmetric Encryption                   | Asymmetric Encryption                        |
| -------------------------------------- | -------------------------------------------- |
| Same key for Encrypt & Decrypt         | Public Key + Private Key                     |
| Faster                                 | Slower                                       |
| Difficult to share secret key securely | Easier key sharing                           |
| Used for bulk data encryption          | Used for secure communication & key exchange |
| Examples: AES, DES, 3DES               | Examples: RSA, Diffie-Hellman, ECC           |
