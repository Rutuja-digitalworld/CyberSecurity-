# 1. Metasploit Framework

### Key Terms

* **Exploit** → Code that takes advantage of a vulnerability.
* **Vulnerability** → Weakness or flaw in a system that can be exploited.
* **Payload** → Code executed on the target after successful exploitation (e.g., shell, command execution).

---

# Metasploit Modules

## 1. Auxiliary

* Used for **scanning, enumeration, fuzzing, crawling**.
* **Does NOT exploit** vulnerabilities.

**Examples:** Scanner, Crawler, Fuzzer

---

## 2. Encoders

* Encode **payloads/exploits**.
* Helps bypass **signature-based antivirus** (limited success).

---

## 3. Evasion

* Attempts to **bypass antivirus/security software**.

---

## 4. Exploits

* Modules that **exploit vulnerabilities** in target systems.

**Examples:** Windows, Linux, Android, Unix exploits.

---

## 5. NOPs (No Operation)

* CPU performs **no action** for one cycle.
* Used as **buffers** to maintain payload size and reliability.

---

## 6. Payloads

* Code executed **after successful exploitation**.

**Examples:**

* Open a shell
* Run a command
* Add a user
* Launch `calc.exe`
* Install malware/backdoor

### Types of Payloads

#### ➡️Adapters

* Convert payloads into different formats.
* Example: Wrap payload into a **PowerShell** command.

#### ➡️Singles (Inline Payloads)

* Self-contained payloads.
* No additional download required.

**Examples:** Add user, launch Notepad.

#### ➡️Stagers

* Establish connection between **Metasploit ↔ Target**.
* Downloads the actual payload (**Stage**).

#### ➡️Stages

* Main payload downloaded by the stager.
* Allows larger payloads.

---

## Single vs Staged Payload

### ➡️Single (Inline) Payload

```text
generic/shell_reverse_tcp
```

* Uses **_ (underscore)**
* Entire payload sent at once.

### ➡️Staged Payload

```text
windows/x64/shell/reverse_tcp
```

* Uses **/ (slash)**
* Sends **Stager first → then downloads Stage**.

> **Memory Trick:**
> **`_` = Single Payload**
> **`/` = Staged Payload**

---

## 7. Post Modules

* Used **after gaining access** to the target.
* Helps in **post-exploitation** activities.

**Examples:**

* Credential dumping
* Privilege escalation
* System enumeration
* Data collection
