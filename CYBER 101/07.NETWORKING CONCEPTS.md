# 1.**OSI Model - Important Notes (Summarized)**

### **OSI Model**

* **OSI (Open Systems Interconnection)** → 7-layer model explaining how data travels in a network.
* **Mnemonic (Bottom → Top):** *Please Do Not Throw Spinach Pizza Away*

| Layer | Name         | Function                                       | Examples                                          |
| ----- | ------------ | ---------------------------------------------- | ------------------------------------------------- |
| **7** | Application  | Provides network services to applications      | HTTP, FTP, DNS, SMTP, POP3, IMAP                  |
| **6** | Presentation | Data encoding, encryption, compression         | Unicode, MIME, JPEG, PNG                          |
| **5** | Session      | Establishes & maintains communication sessions | NFS, RPC                                          |
| **4** | Transport    | End-to-end communication, segmentation         | TCP, UDP                                          |
| **3** | Network      | Routing & logical addressing between networks  | IP, ICMP, IPSec                                   |
| **2** | Data Link    | Communication within the same network          | Ethernet (802.3), Wi-Fi (802.11), **MAC Address** |
| **1** | Physical     | Physical transmission of data                  | Ethernet cable, Fiber, Wireless signals           |

### **Important Points**

* **Layer 1 (Physical):** Uses cables, fiber, or wireless signals.
* **Layer 2 (Data Link):** Uses **MAC Address** (48-bit/6-byte hardware address).
* **Layer 3 (Network):** Uses **IP Address** for routing between different networks.
* **Layer 4 (Transport):** Uses **TCP** (reliable) and **UDP** (fast).
* **Layer 6 (Presentation):** Handles **encryption, compression, encoding**.
* **Layer 7 (Application):** User-facing protocols like **HTTP, FTP, DNS**.

### **Layer 2 (Data Link) – Additional Important Points**

* Provides communication between devices on the **same network**.
* Examples: **Ethernet (802.3)** and **Wi-Fi (802.11)**.
* Uses a **MAC (Media Access Control) Address** for device identification.
* A **MAC Address is 48 bits (6 bytes)** long.
* It is written in **hexadecimal format**, with **6 pairs of hexadecimal digits separated by colons (`:`)**.

  * Example: `A4:C3:F0:85:AC:2D`
* **First 3 bytes (24 bits)** → Identify the **vendor/manufacturer** of the network interface card (NIC).
* **Last 3 bytes (24 bits)** → Unique identifier assigned to that specific network interface.
* Every Ethernet/Wi-Fi frame contains:

  * **Source MAC Address**
  * **Destination MAC Address**

# 2.TCP/IP Model (Transmission Control Protocol / Internet Protocol)

* Developed in the **1970s** by the **U.S. Department of Defense (DoD)**.
* Designed to keep communication working even if parts of the network fail (high reliability).
* Uses **dynamic routing** to find alternate paths when the network topology changes.
* **Practical model** used on the Internet (unlike the conceptual OSI model).

### TCP/IP Layers (Top → Bottom)

1. **Application Layer**

   * Combines **OSI Layers 5, 6 & 7** (Session, Presentation, Application).
   * Provides services directly to user applications.
   * **Protocols:** HTTP, HTTPS, FTP, SMTP, POP3, IMAP, SSH, Telnet.

2. **Transport Layer**

   * Equivalent to **OSI Layer 4**.
   * Provides end-to-end communication, reliability, and error checking.
   * **Protocols:** TCP, UDP.

3. **Internet Layer**

   * Equivalent to **OSI Layer 3 (Network Layer)**.
   * Handles logical addressing and routing between networks.
   * **Protocols:** IP, ICMP, IPSec.

4. **Link Layer**

   * Equivalent to **OSI Layer 2 (Data Link Layer)**.
   * Handles communication within the same local network.
   * **Technologies:** Ethernet (802.3), Wi-Fi (802.11).

### TCP/IP vs OSI

| TCP/IP Layer | OSI Layer(s)    |
| ------------ | --------------- |
| Application  | Layers 5, 6 & 7 |
| Transport    | Layer 4         |
| Internet     | Layer 3         |
| Link         | Layer 2         |

### Important Note

* Many modern books use a **5-layer TCP/IP model** by adding the **Physical Layer** separately:

  * Application
  * Transport
  * Network
  * Link
  * Physical

# 3.**IP Address and Subnets(IPv4)**

* Every device on a network requires a **unique IP address** for communication.
* **IPv4 (Internet Protocol Version 4)** is the most commonly used IP addressing system.
* An IPv4 address consists of **4 octets (4 bytes = 32 bits)**.
* Each octet ranges from **0 to 255**.
* **Example:** `192.168.1.10`
* **Network Address:** First address of a subnet (e.g., `192.168.1.0`)
* **Broadcast Address:** Last address of a subnet (e.g., `192.168.1.255`) – used to send data to **all devices** on that network.
* Maximum possible IPv4 addresses ≈ **2³² (about 4.3 billion)**.
* **Subnet Mask** (e.g., `255.255.255.0` or `/24`) determines the network and host portions of an IP address.
* **/24** means the first **24 bits** identify the network, leaving the remaining bits for host addresses.
* In a `/24` network, usable host IPs range from **192.168.1.1 – 192.168.1.254**.

### **Viewing IP Configuration**

* **Windows:** `ipconfig`
* **Linux:** `ifconfig` or `ip a s`

### **Private IP Address Ranges (RFC 1918)**

* **10.0.0.0 – 10.255.255.255** (`10/8`)
* **172.16.0.0 – 172.31.255.255** (`172.16/12`)
* **192.168.0.0 – 192.168.255.255** (`192.168/16`)

### **Public vs Private IP**

* **Public IP:** Accessible over the Internet; globally unique.
* **Private IP:** Used within local networks; not directly accessible from the Internet.
* Private IPs access the Internet through **NAT (Network Address Translation)**.

### **Router**

* Works at **Layer 3 (Network Layer)**.
* Reads the **destination IP address** and forwards packets to the best path.
* Connects different networks and enables communication across the Internet.

### **Private IPv4 Ranges**

* **10.0.0.0 – 10.255.255.255** → `10.x.x.x`
* **172.16.0.0 – 172.31.255.255** → `172.16–31.x.x`
* **192.168.0.0 – 192.168.255.255** → `192.168.x.x`

**Rule:** Any IP outside these ranges is **Public**.

---

### **Valid IPv4 Rule**

* Must contain **4 octets** (`A.B.C.D`).
* Each octet must be **0–255**.

**Example:** `192.168.305.19` ❌ (305 > 255)

# 4.**UDP & TCP** 

## **Transport Layer (Layer 4)**

* Used for **process-to-process communication** between hosts.
* Two main protocols:

  * **UDP (User Datagram Protocol)**
  * **TCP (Transmission Control Protocol)**

---

## **UDP (User Datagram Protocol)**

* **Connectionless** protocol (no connection setup required).
* **Faster** but **less reliable**.
* No acknowledgment or guarantee of packet delivery.
* Best for **speed-sensitive applications**.

**Key Points**

* Uses **Port Numbers (1–65535)** to identify applications.
* Does **not** check if packets are received.
* Similar to **sending normal mail without delivery confirmation**.

**Examples**

* DNS
* Online gaming
* Live video/audio streaming
* VoIP

---

## **TCP (Transmission Control Protocol)**

* **Connection-oriented** protocol.
* **Reliable** communication with error checking.
* Ensures data reaches the destination correctly and in order.
* Slower than UDP due to reliability mechanisms.

**Key Points**

* Uses **Sequence Numbers** to track packets.
* Uses **Acknowledgment (ACK)** to confirm received data.
* Retransmits lost packets.

**Examples**

* HTTP/HTTPS
* FTP
* Email (SMTP, IMAP, POP3)
* SSH

---

## **TCP Three-Way Handshake** 

Two flags are used: SYN (Synchronise) and ACK (Acknowledgment). The packets are sent as follows:
* **SYN:** Client sends a connection request to the server.
* **SYN-ACK:** Server acknowledges the request and responds with its own connection request.
* **ACK:** Client acknowledges the server's response, and the connection is established.

## **Port Numbers**

* Used to identify **specific applications/processes**.
* Range: **1 – 65535** (Port **0** is reserved).
* Total ports ≈ **65,000**.

---

## **UDP vs TCP**

| UDP                             | TCP                                         |
| ------------------------------- | ------------------------------------------- |
| Connectionless                  | Connection-oriented                         |
| Faster                          | Slower                                      |
| No acknowledgment               | Acknowledgment available                    |
| No delivery guarantee           | Reliable delivery                           |
| No error recovery               | Error detection & retransmission            |
| Used for streaming, gaming, DNS | Used for web browsing, email, file transfer |

# 5.Encapsulation

* **Encapsulation** is the process where **each network layer adds its own header** (and sometimes a **trailer**) to the data before passing it to the next lower layer.
* Each layer performs its own function independently.

## Encapsulation Process

1. **Application Data**

   * User creates data (e.g., email, message, web request).

2. **Transport Layer**

   * Adds **TCP/UDP header**.
   * Creates:

     * **TCP → Segment**
     * **UDP → Datagram**

3. **Network (Internet) Layer**

   * Adds **IP header**.
   * Creates an **IP Packet**.

4. **Data Link Layer**

   * Adds **Header + Trailer**.
   * Creates a **Frame** (Ethernet/WiFi).

### Encapsulation Flow

```text
Application Data
        ↓
TCP Segment / UDP Datagram
        ↓
IP Packet
        ↓
Ethernet/WiFi Frame
```

> **Note:** At the receiver's end, the process is reversed (**Decapsulation**) until the original application data is extracted.

---

# Life of a Packet

1. User sends a request (e.g., searches on TryHackMe).
2. **Application Layer** creates an **HTTP/HTTPS request**.
3. **Transport Layer (TCP)** establishes a **Three-Way Handshake** and sends the request as **TCP Segments**.
4. **Network Layer (IP)** adds **Source IP** and **Destination IP** to create an **IP Packet**.
5. **Data Link Layer** adds **Header + Trailer** to create a **Frame**.
6. **Routers** forward the packet by checking the **Destination IP** until it reaches the target network.
7. At the destination, **Decapsulation** removes headers layer by layer and delivers the original data to the application.

---

## Important Terms

* **TCP Data Unit:** **Segment**
* **UDP Data Unit:** **Datagram**
* **IP Data Unit:** **Packet**
* **Data Link Data Unit:** **Frame**
* **Process of Adding Headers:** **Encapsulation**
* **Process of Removing Headers:** **Decapsulation**

# 6.TELNET (Teletype Network)

* **TELNET** = **Teletype Network**
* A **network protocol** used for **remote terminal connections**.
* Allows users to **connect to a remote system** and execute **text-based commands**.
* Works over **TCP**.
* Can connect to **any service listening on a TCP port**.
* **Default Port:** **23**
* **Not secure** (data, including passwords, is sent in **plain text**).

---

# Common Services Used in Lab

| Service         | Default Port | Purpose                             |
| --------------- | :----------: | ----------------------------------- |
| Echo Server     |     **7**    | Returns (echoes) whatever you type. |
| Daytime Server  |    **13**    | Displays the current date and time. |
| HTTP/Web Server |    **80**    | Serves web pages.                   |

> **Note:** Echo and Daytime servers are considered **security risks** and are rarely enabled today.

---

# Telnet Syntax

```bash
telnet <IP_Address> <Port>
```

### Examples

```bash
telnet MACHINE_IP 7
```

Connect to Echo Server.

```bash
telnet MACHINE_IP 13
```

Connect to Daytime Server.

```bash
telnet MACHINE_IP 80
```

Connect to HTTP Server.

After connecting to **Port 80**, type:

```http

```

# Important Points

* **TELNET** is mainly used for:

  * Remote administration (legacy)
  * Testing TCP services
  * Troubleshooting network connections

* **Uses TCP**

* **Default Port:** **23**

* **No encryption** → Not secure

* **Replaced by SSH** for secure remote access
