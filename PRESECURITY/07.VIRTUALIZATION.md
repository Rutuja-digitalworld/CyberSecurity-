**Virtualization introduced a new idea:**<br>
 multiple applications share the same physical server safely<br>
A virtualization layer, called a hypervisor, was introduced to act as a referee between lab machines and allow each virtual computer to behave independently, like a physical computer.<br>

**Virtualization (Building Analogy)**<br>
•Virtualization allows multiple virtual computers (VMs) to run on a single physical computer.<br>
•Each VM works independently but shares the same physical hardware.<br>

The key benefits of virtualization are:<br>
•Cost savings<br>
•Better resource usage<br>
•Safe testing for cyber security<br>
•Faster deployment<br>
•Flexibility<br>
•Portability<br>
•Scalability<br>
•Centralized Management<br>
•Each VM has its own:<br>
➡Operating System<br>
➡Applications<br>
➡Settings<br>
➡Storage<br>
**Hypervisor**<br>
•Software that creates and manages VMs.<br>
•Ensures VMs are isolated and share resources safely.<br>
•It divides a physical computer into multiple virtual computers.<br>
•Allocates CPU, RAM, and Storage to each VM.<br>
•Keeps VMs isolated and secure.<br>
•Manages VM operations: Start, Stop, Pause, Clone, Delete.<br>

**Types of Hypervisors**<br>
*Type 1 (Bare Metal)*<br>
•Runs directly on hardware.<br>
•Faster and more efficient.<br>
•Used in servers and data centers.<br>

*Type 2 (Hosted)*<br>
•Runs on top of an existing Operating System.<br>
•Easier to install and use.<br>
•Ideal for learning and testing.<br>

**Lab Machine (VM)**<br>
•A Lab Machine (VM) is a virtual computer created by a hypervisor.<br>
•Has its own virtual CPU, RAM, Storage, and Network.<br>
•Can run any OS (Windows, Linux, Kali, etc.).<br>
•Completely isolated from other VMs.<br>
•If one VM crashes, others continue working.<br>
•VM Software<br>
➡Oracle VirtualBox<br>
➡VMware Workstation<br>
**Uses**<br>
✅ Run a different OS without buying another computer<br>
✅ Test suspicious/malicious files safely<br>

**Containers**<br>
•A Container is a lightweight environment that runs a single application and its dependencies.<br>
•Shares the host OS kernel (doesn't need a full OS).<br>
•Starts faster and uses fewer resources than a VM.<br>
•Containers must match the host OS type.<br>
*Features*<br>
✅ Packages app + dependencies<br>
✅ Fast startup<br>
✅ Resource efficient<br>
✅ Isolated from other containers<br>
✅ Consistent across different machines<br>

**Docker**<br>
Docker is the most popular platform for creating and managing containers.<br>
