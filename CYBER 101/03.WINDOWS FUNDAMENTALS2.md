# 1.SYSTEM CONFIGURATION 
The System Configuration utility (MSConfig) is for advanced troubleshooting, and its main purpose is to help diagnose startup issues. 

➡The utility has five tabs across the top. Below are the names for each tab. We will briefly cover each tab in this task. 

1.General

2.Boot

3.Services

4.Startup

5.Tools

➡**General Tab** – Selects which devices and services Windows loads during startup (Normal, Diagnostic, or Selective startup).

➡**Boot Tab** – Configures operating system boot options and startup settings.

* **Services Tab** – Displays all system services (running or stopped); services are background applications that run without user interaction.

**Startup Tab (MSConfig)** → Displays programs that automatically launch when Windows starts and lets you enable/disable startup apps to improve boot performance.

**Startup Management (Windows Server)**

* **MSConfig Startup Tab** → Not used for managing startup programs; Microsoft recommends using **Task Manager** instead.
* **Windows Server Difference** → Startup apps usually do **not appear** in Task Manager or MSConfig Startup tab.
* **View Startup Programs** → Press **Win + R → type `shell:startup` → Enter**.
* **Startup Folder** → Contains shortcuts/executables that run automatically when a user logs in.
* **Purpose** → Used to verify and manage user-level startup applications on Windows Server.

### MSConfig & Advanced System Settings (Short Notes)

* **Tools Tab (MSConfig)** → Lists Windows utilities/tools with brief descriptions for system configuration.

---------

# Advanced System Settings

* **Advanced System Settings** → Extra options to manage performance and system recovery.
* **Performance Settings** → Controls system performance and virtual memory (page file).
* **Page File** → Uses disk space as extra RAM when physical memory is full.
* **Page File Details** → Shows drive location, initial size, maximum size, and whether Windows manages it automatically.
* **Startup and Recovery** → Configures system startup behavior and recovery options after crashes.
* **Crash Dump File** → Stores error details after a system crash (e.g., BSOD) for troubleshooting and analysis.

  ### Startup & Recovery – Crash Dump (Short Notes)

* **Write Debugging Information** → Selects the type of crash dump Windows saves after a system crash.
* **Crash Dump Types**:

  * **Automatic Memory Dump** → Windows automatically manages dump size.
  * **Kernel Memory Dump** → Saves only kernel memory.
  * **Small Memory Dump (256 KB)** → Saves basic crash information.
  * **Complete Memory Dump** → Saves all system memory.
  * **None** → No crash dump is created.
* **Purpose** → Crash dumps help diagnose and troubleshoot system crashes (e.g., BSOD).

### Important Commands / Answers

* **Service by Sysinternals** → **PsShutdown**
* **Windows License Registered To** → **Windows User**
* **Windows Troubleshooting Command** → `C:\Windows\System32\control.exe /name Microsoft.Troubleshooting`
* **Open Control Panel (.exe)** → `control.exe`

# 2.User Account Control (UAC) – Short Notes

* **UAC** → Protects the system by prompting before administrator-level changes.
* **UAC Levels**:

  * **Always Notify** → Alerts for all changes (highest security).
  * **Notify for Apps** → Alerts only when apps make changes (**default**).
  * **Notify without Dimming** → Same as above, but screen doesn't dim.
  * **Never Notify** → No alerts (not recommended).
* **UAC Settings Command** → `UserAccountControlSettings.exe`

# 3.Computer Management (compmgmt) – Short Notes

* **Computer Management** → Includes **System Tools, Storage, and Services & Applications**.

### Task Scheduler

* **Task Scheduler** → Automates tasks (apps/scripts) at specified times or events.
* **Task Triggers** → Can run at **logon, logoff, scheduled time, or one-time event**.
* **Task Scheduler Library** → Displays all scheduled tasks and their details.
* **Create Basic Task** → Used to create a new scheduled task.

### Event Viewer

* **Event Viewer** → Records system events/logs for monitoring, troubleshooting, and security investigations.
* **Left Pane** → Lists event log providers.
* **Middle Pane** → Shows event summary and details.
* **Right Pane** → Contains actions/options for managing logs.

# 4.System Information (msinfo32) – 

* **Purpose:** Displays detailed information about the computer's **hardware, system components, and software** for troubleshooting.
* **Command:** `msinfo32.exe`

### **Main Sections**

* **System Summary:** General system details (Processor, BIOS, System Name, RAM, etc.)
* **Hardware Resources:** Information about hardware resources (DMA, IRQs, Memory).
* **Components:** Details of installed hardware (Display, Storage, Network, Input devices).
* **Software Environment:** Information about installed software, drivers, services, environment variables, and network connections.

### **Environment Variables**

* Store system information like Windows path, processor details, and temporary folder locations.
* Example: **`%ComSpec% = %SystemRoot%\system32\cmd.exe`**
* Can also be accessed via **Advanced System Settings → Environment Variables**.

### **COMMANDS**

* **Open System Information:** `msinfo32.exe`
* **ComSpec Value:** `%SystemRoot%\system32\cmd.exe`

  # 5.Resource Monitor (resmon) – Important Notes**

* **Purpose:** Monitors real-time **CPU, Memory, Disk, and Network** usage for troubleshooting.
* **Command:** `resmon.exe`

### **Main Sections**

* **CPU:** Processor usage by processes.
* **Memory:** RAM usage and allocation.
* **Disk:** Disk activity and read/write operations.
* **Network:** Network usage and active connections.

### **Features**

* View resource usage per process.
* Filter data for specific applications/processes.
* Start, stop, pause, or resume services.
* Close unresponsive applications.
* Detect file-locking conflicts and deadlocked processes.
* Real-time graphs for all resources.

# 6.**Command Prompt (cmd) – Important Notes**

* **Purpose:** Text-based interface to interact with Windows using commands.
* **Command:** `cmd.exe`

### **Common Commands**

| **Command**     | **Purpose**                                                        |
| --------------- | ------------------------------------------------------------------ |
| `hostname`      | Displays the computer name.                                        |
| `whoami`        | Displays the currently logged-in user.                             |
| `ipconfig`      | Shows network/IP configuration.                                    |
| `ipconfig /all` | Displays detailed network information.                             |
| `ipconfig /?`   | Shows help for the `ipconfig` command.                             |
| `cls`           | Clears the Command Prompt screen.                                  |
| `netstat`       | Displays active TCP/IP connections and network statistics.         |
| `net`           | Manages network resources (users, shares, groups, sessions, etc.). |
| `net help`      | Displays help for `net` commands.                                  |
| `net help user` | Shows help for the `net user` command.                             |

### COMMANDS:

* **Full command for Internet Protocol Configuration:**
  `C:\Windows\System32\cmd.exe /k %windir%\system32\ipconfig.exe`
* **Detailed network information:**
  `ipconfig /all`

# 7.**Windows Registry – Important Notes**

* **Purpose:** Central database that stores Windows configuration settings for the OS, users, applications, and hardware.
* **Command:** `regedt32.exe`

### **Registry Stores**

* User profiles
* Installed applications
* Folder & application settings
* Hardware configuration
* Port information

> **Note:** Editing the registry incorrectly can cause system issues, so it should only be modified by advanced users.
