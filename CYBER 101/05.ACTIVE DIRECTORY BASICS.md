# **Windows Domain & Active Directory (AD) – Short Notes**

* **Windows Domain:** A group of users and computers managed **centrally** by an organization.
* **Active Directory (AD):** Central repository for managing users, computers, and security policies.
* **Domain Controller (DC):** Server that runs Active Directory services.

**Advantages:**

* **Centralized Identity Management:** Manage all users from one place.
* **Centralized Security Policies:** Apply security settings to all users and computers.

**Real-World Example:**

* School/work login credentials work on any domain computer because authentication is handled by **Active Directory**.
* AD also enforces restrictions (e.g., no Control Panel or admin access).

# **Active Directory Domain Services (AD DS) – Short Notes**

### **AD DS**

* **AD DS (Active Directory Domain Services):** Central directory that stores and manages all network objects (users, groups, computers, printers, etc.).

### **Users**

* **Security Principal:** Can be authenticated and assigned permissions.
* **Types:**

  * **User Accounts:** Represent employees.
  * **Service Accounts:** Used by services (IIS, MSSQL) with limited privileges.

### **Machines (Computers)**

* Every domain-joined computer gets a **Machine Account**.
* Machine accounts are also **Security Principals**.
* **Naming:** `ComputerName$` (e.g., `DC01$`).
* **Password:** Automatically rotated (120 random characters).

### **Security Groups**

* Used to **assign permissions** to users and computers.
* Can contain **users, computers, and other groups**.

### **Important Default Groups**

* **Domain Admins:** Full control over the domain.
* **Server Operators:** Manage Domain Controllers.
* **Backup Operators:** Access all files for backups.
* **Account Operators:** Create/modify user accounts.
* **Domain Users:** All domain users.
* **Domain Computers:** All domain computers.
* **Domain Controllers:** All DCs.

### **Active Directory Users and Computers (ADUC)**

* Tool used to **create, delete, modify users, groups, computers, and reset passwords**.

### **Organizational Units (OUs)**

* Container used to **organize users and computers**.
* Mainly used to **apply Group Policies**.
* A user/computer can belong to **only one OU**.

### **Default AD Containers**

* **Builtin:** Default Windows groups.
* **Computers:** Newly joined computers.
* **Domain Controllers:** Stores DCs.
* **Users:** Default users and groups.
* **Managed Service Accounts:** Service accounts.

### **OU vs Security Group**

| **OU (Organizational Unit)**     | **Security Group**                     |
| -------------------------------- | -------------------------------------- |
| Used to apply **Group Policies** | Used to assign **permissions**         |
| User belongs to **one OU**       | User can belong to **multiple groups** |
| Organizes users/computers        | Controls access to resources           |

# **Managing AD OUs & Users – Short Notes**

### **Deleting OUs**

* OUs are **protected from accidental deletion** by default.
* Enable **View → Advanced Features**.
* Go to **OU → Properties → Object** → Uncheck **Protect object from accidental deletion**.
* Delete the OU (all child users/groups/OUs are also deleted).

### **Managing Users**

* Create/Delete users to match the organization's structure.

### **Delegation**

* **Delegation:** Grant specific users permissions over an OU without making them Domain Admins.
* Common use: Allow IT staff to **reset users' passwords**.

### **Delegate Control Steps**

1. Right-click **OU → Delegate Control**.
2. Add the user (e.g., **Phillip**).
3. Choose **Reset user passwords and force password change at next logon**.

### **PowerShell Commands**

* **Reset Password:**

```powershell
Set-ADAccountPassword sophie -Reset -NewPassword (Read-Host -AsSecureString -Prompt 'New Password')
```

* **Force Password Change at Next Login:**

```powershell
Set-ADUser -ChangePasswordAtLogon $true -Identity sophie
```

### **Important**

* **Delegation ≠ Domain Admin** (limited privileges only).
* Phillip can reset passwords only for users in delegated OUs (e.g., Sales).

# **Managing Computers in Active Directory – Short Notes**

### **Computers Container**

* By default, all **domain-joined computers** (except Domain Controllers) are placed in the **Computers** container.

### **Recommended OU Structure**

* **Workstations:** User PCs and laptops.
* **Servers:** Machines providing network services.
* **Domain Controllers:** Managed in a separate default OU by Windows.

### **Why Separate OUs?**

* Apply **different Group Policies (GPOs)** to workstations and servers.
* Improves **security, management, and organization**.

### **Organization Steps**

* Create **Workstations** and **Servers** OUs under the **thm.local** domain.
* Move:

  * **PCs & Laptops → Workstations OU**
  * **Servers → Servers OU**

### **Key Points**

* **Workstations:** Used by employees; avoid logging in with privileged accounts.
* **Servers:** Provide services to users and systems.
* **Domain Controllers:** Most critical systems; manage Active Directory and store user password hashes.

# **Active Directory (AD) – Group Policies (GPO) | Short Notes**

### **Group Policy Object (GPO)**

* Collection of settings used to configure **users and computers** in a domain.
* Applied to **Organizational Units (OUs)**.
* Helps enforce **security policies and system configurations**.

### **Group Policy Management (GPMC)**

* Tool used to **create, edit, link, and manage GPOs**.
* GPOs are created under **Group Policy Objects** and then linked to the required OU.

### **Default GPOs**

* **Default Domain Policy** → Applies to the entire domain (password policies, account lockout, etc.).
* **Default Domain Controllers Policy** → Applies only to Domain Controllers.
* **RDP Policy** → Domain-wide Remote Desktop settings.

### **GPO Scope**

* A GPO affects the **linked OU** and all its **child OUs**.
* **Security Filtering** allows applying a GPO only to selected users/groups (default: **Authenticated Users**).

### **Editing GPOs**

* Path: **Computer/User Configuration → Policies → Windows Settings → Security Settings**
* Example: Set **Minimum Password Length = 10 characters**.

### **SYSVOL**

* **SYSVOL** is the network share used to distribute GPOs.
* Default path:

  ```
  C:\Windows\SYSVOL\sysvol\
  ```

### **Update Group Policies**

* Changes may take **up to 2 hours** to apply.
* Force update:

  ```powershell
  gpupdate /force
  ```

# **Exam Points**

* **GPO** = Applies policies to **users & computers**.
* **gpupdate /force** = Force immediate policy update.
* **Password policies** are usually configured in the **Default Domain Policy**.
* **Computer Configuration** affects computers.
* **User Configuration** affects users.
