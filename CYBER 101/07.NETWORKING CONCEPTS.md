# **OSI Model - Important Notes (Summarized)**

### **OSI Model**

* **OSI (Open Systems Interconnection)** → 7-layer model explaining how data travels in a network.
* **Mnemonic (Bottom → Top):** *Please Do Not Throw Spinach Pizza Away*

| Layer | Name         | Function                                       | Examples                                          |
| ----- | ------------ | ---------------------------------------------- | ------------------------------------------------- |
| **7** | Application  | Provides network services to applications      | HTTP, FTP, DNS, SMTP, POP3, IMAP                  |
| **6** | Presentation | Data encoding, encryption, compression         | Unicode, MIME, JPEG, PNG                          |
| **5** | Session      | Establishes & maintains communication sessions | NFS, RPC                                          |
| **4** | Transport    | End-to-end communication, segmentation         | TCP, UDP                                          |
| **3** | Network      | Routing & logical addressing between networks  | IP, ICMP, IPSec                                   |
| **2** | Data Link    | Communication within the same network          | Ethernet (802.3), Wi-Fi (802.11), **MAC Address** |
| **1** | Physical     | Physical transmission of data                  | Ethernet cable, Fiber, Wireless signals           |

### **Important Points**

* **Layer 1 (Physical):** Uses cables, fiber, or wireless signals.
* **Layer 2 (Data Link):** Uses **MAC Address** (48-bit/6-byte hardware address).
* **Layer 3 (Network):** Uses **IP Address** for routing between different networks.
* **Layer 4 (Transport):** Uses **TCP** (reliable) and **UDP** (fast).
* **Layer 6 (Presentation):** Handles **encryption, compression, encoding**.
* **Layer 7 (Application):** User-facing protocols like **HTTP, FTP, DNS**.

### **Layer 2 (Data Link) – Additional Important Points**

* Provides communication between devices on the **same network**.
* Examples: **Ethernet (802.3)** and **Wi-Fi (802.11)**.
* Uses a **MAC (Media Access Control) Address** for device identification.
* A **MAC Address is 48 bits (6 bytes)** long.
* It is written in **hexadecimal format**, with **6 pairs of hexadecimal digits separated by colons (`:`)**.

  * Example: `A4:C3:F0:85:AC:2D`
* **First 3 bytes (24 bits)** → Identify the **vendor/manufacturer** of the network interface card (NIC).
* **Last 3 bytes (24 bits)** → Unique identifier assigned to that specific network interface.
* Every Ethernet/Wi-Fi frame contains:

  * **Source MAC Address**
  * **Destination MAC Address**

# 2.TCP/IP Model (Transmission Control Protocol / Internet Protocol)

* Developed in the **1970s** by the **U.S. Department of Defense (DoD)**.
* Designed to keep communication working even if parts of the network fail (high reliability).
* Uses **dynamic routing** to find alternate paths when the network topology changes.
* **Practical model** used on the Internet (unlike the conceptual OSI model).

### TCP/IP Layers (Top → Bottom)

1. **Application Layer**

   * Combines **OSI Layers 5, 6 & 7** (Session, Presentation, Application).
   * Provides services directly to user applications.
   * **Protocols:** HTTP, HTTPS, FTP, SMTP, POP3, IMAP, SSH, Telnet.

2. **Transport Layer**

   * Equivalent to **OSI Layer 4**.
   * Provides end-to-end communication, reliability, and error checking.
   * **Protocols:** TCP, UDP.

3. **Internet Layer**

   * Equivalent to **OSI Layer 3 (Network Layer)**.
   * Handles logical addressing and routing between networks.
   * **Protocols:** IP, ICMP, IPSec.

4. **Link Layer**

   * Equivalent to **OSI Layer 2 (Data Link Layer)**.
   * Handles communication within the same local network.
   * **Technologies:** Ethernet (802.3), Wi-Fi (802.11).

### TCP/IP vs OSI

| TCP/IP Layer | OSI Layer(s)    |
| ------------ | --------------- |
| Application  | Layers 5, 6 & 7 |
| Transport    | Layer 4         |
| Internet     | Layer 3         |
| Link         | Layer 2         |

### Important Note

* Many modern books use a **5-layer TCP/IP model** by adding the **Physical Layer** separately:

  * Application
  * Transport
  * Network
  * Link
  * Physical

# 3.**IP Address and Subnets(IPv4)**

* Every device on a network requires a **unique IP address** for communication.
* **IPv4 (Internet Protocol Version 4)** is the most commonly used IP addressing system.
* An IPv4 address consists of **4 octets (4 bytes = 32 bits)**.
* Each octet ranges from **0 to 255**.
* **Example:** `192.168.1.10`
* **Network Address:** First address of a subnet (e.g., `192.168.1.0`)
* **Broadcast Address:** Last address of a subnet (e.g., `192.168.1.255`) – used to send data to **all devices** on that network.
* Maximum possible IPv4 addresses ≈ **2³² (about 4.3 billion)**.
* **Subnet Mask** (e.g., `255.255.255.0` or `/24`) determines the network and host portions of an IP address.
* **/24** means the first **24 bits** identify the network, leaving the remaining bits for host addresses.
* In a `/24` network, usable host IPs range from **192.168.1.1 – 192.168.1.254**.

### **Viewing IP Configuration**

* **Windows:** `ipconfig`
* **Linux:** `ifconfig` or `ip a s`

### **Private IP Address Ranges (RFC 1918)**

* **10.0.0.0 – 10.255.255.255** (`10/8`)
* **172.16.0.0 – 172.31.255.255** (`172.16/12`)
* **192.168.0.0 – 192.168.255.255** (`192.168/16`)

### **Public vs Private IP**

* **Public IP:** Accessible over the Internet; globally unique.
* **Private IP:** Used within local networks; not directly accessible from the Internet.
* Private IPs access the Internet through **NAT (Network Address Translation)**.

### **Router**

* Works at **Layer 3 (Network Layer)**.
* Reads the **destination IP address** and forwards packets to the best path.
* Connects different networks and enables communication across the Internet.

### **Private IPv4 Ranges**

* **10.0.0.0 – 10.255.255.255** → `10.x.x.x`
* **172.16.0.0 – 172.31.255.255** → `172.16–31.x.x`
* **192.168.0.0 – 192.168.255.255** → `192.168.x.x`

**Rule:** Any IP outside these ranges is **Public**.

---

### **Valid IPv4 Rule**

* Must contain **4 octets** (`A.B.C.D`).
* Each octet must be **0–255**.

**Example:** `192.168.305.19` ❌ (305 > 255)
